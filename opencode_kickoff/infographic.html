Durable 状态模型

Session.Info + MessageV2.Part 构成真实执行状态；核心索引在本地 SQLite，补充 diff/blob 落在 Storage JSON

📦
Session.Info
session/index.ts
执行边界容器。定义工作目录、workspace、父子关系、权限、summary、share、revert 锚点。不是聊天容器，而是运行时宿主。
directoryworkspaceIDpermissionrevertparentID
🦩
MessageV2.Part
session/message-v2.ts
最小状态单元。text、reasoning、tool、step、patch、subtask、compaction 都是同一层 Part，共享统一写路径。
textreasoningtoolsteppatchsubtaskcompaction
🗄
SQLite + Storage
opencode.db · Drizzle ORM
session/message/part 等核心索引在 SQLite；summary diff 等补充材料通过 Storage 写入文件系统。无外部数据库依赖。
SessionTableMessageTablePartTablesession_diff
📡
Bus 事件投影
bus/index.ts · /event SSE
`updateMessage`/`updatePart` 会先落库再发事件；`updatePartDelta` 只发流式增量，不落库。`/event` 与 `/global/event` 再把这些事件转成 SSE。

高级能力：为什么没有长歪

subagent、compaction、structured output 都接回同一条 durable history

👶
Subagent
不是 persona 切换，而是 child session。TaskTool 通过 Session.create() 创建子 session，走同一主链，共享 runtime 但隔离状态和权限。
📦
Compaction
不是后台 GC，而是 显式写回历史。runtime 会插入 `compaction` part，再由隐藏 agent 生成 summary；旧历史保留，只是在回放时折叠。
🎯
Structured Output
不是 prompt 约束，而是 tool 协议。动态注入 structured output tool，强制 toolChoice=required，模型必须完成工具调用才停机。
